When sleep would waste the run

Close the lid without killing the work.

Long builds

Start the compile, plug in, shut the lid, and come back to results instead of a sleeping Mac.

Large downloads

Let big downloads finish without leaving the display open on your desk.

Remote jobs

Keep SSH sessions, scripts, and remote tasks alive while the Mac is charging.

Backups

Give Time Machine and local backup scripts time to finish cleanly.

Manual install, no surprises

A manual install with the approval steps up front.

This is a small technical release, not an App Store flow. Review the source on GitHub, download the DMG from Releases, then approve the local helper only when macOS asks.

  1. 1

    Download from GitHub Releases

    Grab the DMG from Releases. Source, docs, and packaging scripts are public in the same GitHub repo.

  2. 2

    Approve first launch

    If Gatekeeper blocks the first launch, approve it in macOS Security settings.

  3. 3

    Install the helper

    Admin permission is requested only to install, restore, or remove the local sleep helper.

  4. 4

    Restore or remove

    Use the menu bar controls to turn it off, restore defaults, or uninstall cleanly.

Trust through control

Know what changes. Undo it fast.

LidSwitch has one job: prevent lid-close sleep while plugged in. Battery behavior stays normal unless you opt in, and the restore and uninstall controls are built into the menu.

No credentials or telemetry

It does not collect, transmit, or store passwords, API keys, tokens, account identifiers, personal analytics, or telemetry.

Source is public

Battery stays opt-in

Battery lid-close sleep stays allowed unless you deliberately enable it.

Restore is not hidden
